# BPMN flow validation (bpmnlint) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add bpmnlint validation to the editor — error/warning markers on elements + the built-in "X Errors, Y Warnings" toggle badge — using `bpmnlint:recommended`, with the config packed for Vite via `bpmnlint-pack-config`.

**Architecture:** `bpmn-js-bpmnlint` is a bpmn-js module that ships the whole linting UI. We pack `.bpmnlintrc` (recommended rules) into an importable JS module with `bpmnlint-pack-config` (no webpack loader needed), add the lint module + `linting` config to the parameterized `createBpmnModeler`, import the CSS, and activate linting on mount. Non-destructive (overlays only).

**Tech Stack:** existing Vite + Vitest + bpmn-js 18; new deps `bpmn-js-bpmnlint`, `bpmnlint`, `bpmnlint-pack-config` (dev).

## Global Constraints

- Ruleset: `bpmnlint:recommended` via `.bpmnlintrc` = `{ "extends": "bpmnlint:recommended" }`.
- Pack the config with `bpmnlint-pack-config` into `src/linting/bpmnlintConfig.js` (generated AND committed, so build/packager need no prebuild); npm script `lint:pack` regenerates it.
- Wire via the existing `selectedModuleKeys`/`createBpmnModeler` in `src/editor.ts`: add `"lint"` to the always-on set, add the lint module to `additionalModules`, set `config.linting = { bpmnlint: <packed> }`.
- Linting **active by default** (markers + badge visible on open); the module's badge toggles it.
- Non-destructive: never writes to the `.bpmn`.
- Empirical bits to verify at implementation time (the packed-config module's export shape, the lint module's export, and the `linting` service's activation method on bpmn-js 18) — inspect the real artifact and adapt; gate is build + manual.
- Gates after each task: `npm test`, `npm run typecheck`, `npm run build` green (119 baseline).
